## 2. ビルド方法

このプログラムは CMake を使ってビルドします。事前に [CMake 3.15 以上](https://cmake.org/) をインストールしてください。

### Windows (Visual Studio 2022)

1. [CMake](https://cmake.org/) と [Visual Studio 2022](https://visualstudio.microsoft.com/) をインストールしてください。
2. リポジトリのルートで以下のコマンドを実行してください。

```bat
cmake -S . -B build -G "Visual Studio 17 2022"
```

3. 生成された `build\rosslers.sln` を Visual Studio 2022 で開き、ビルドしてください。  
   または、コマンドラインでビルドすることもできます。

```bat
cmake --build build --config Release
```

4. ビルドが完了すると `build\Release\rosslers.exe` が生成されます。  
   実行には同じディレクトリに `freeglut.dll` が必要ですが、CMake が自動的にコピーします。

### macOS (Xcode)

1. Xcode と [CMake](https://cmake.org/) をインストールしてください。  
   Xcode のコマンドラインツールも必要です。

```bash
xcode-select --install
```

2. リポジトリのルートで以下のコマンドを実行してください。

```bash
cmake -S . -B build -G Xcode
```

3. 生成された `build/rosslers.xcodeproj` を Xcode 16 で開き、ビルドしてください。  
   または、コマンドラインでビルドすることもできます。

```bash
cmake --build build --config Release
```

4. ビルドが完了すると `build/Release/rosslers` が生成されます。

### Ubuntu Linux (Makefile)

1. 必要なパッケージをインストールしてください。

```bash
sudo apt update
sudo apt install cmake build-essential freeglut3-dev libglu1-mesa-dev
```

2. リポジトリのルートで以下のコマンドを実行してください。

```bash
cmake -S . -B build -G "Unix Makefiles"
cmake --build build
```

3. ビルドが完了すると `build/rosslers` が生成されます。

## 3. 使用方法

### 3.1 起動

ビルド後に生成された実行ファイル（`rosslers.exe` または `rosslers`）を実行してください。起動するとウィンドウが開き、Rössler アトラクタの解曲線上を紙飛行機がアニメーションします。

### 3.2 キーボード操作

| キー | 動作 |
| ------ | ------ |
| `g` | アニメーションを再開する |
| `s` | アニメーションを一時停止する |
| `スペース` | 一時停止中に 1 コマ進める |
| `v` | 視点を順方向に切り替える（0→1→2→3→0） |
| `V` | 視点を逆方向に切り替える（0→3→2→1→0） |
| `u` | 視点の高さを上げる（視点 1〜3 有効） |
| `d` | 視点の高さを下げる（視点 1〜3 有効） |
| `+` | 視点 1（追従視点）でズームイン（紙飛行機に近づく） |
| `-` | 視点 1（追従視点）でズームアウト（紙飛行機から遠ざかる） |
| `t` | 解曲線の表示を「帯（Lane）」と「軌跡（Tail）」で切り替える |
| `ESC` | プログラムを終了する |

### 3.3 マウス操作

マウスの**右ボタン**をクリックするとコンテキストメニューが表示されます。

**View サブメニュー**：視点モードを選択します。

| 項目 | 説明 |
| ------ | ------ |
| Full view | 真上からの固定俯瞰視点 |
| Follow the Plane | 紙飛行機の斜め後方から追従する視点（デフォルト） |
| From the Center | アトラクタ中心から紙飛行機方向を見る視点 |
| From Outside | 原点方向から外側へ向けた視点 |
| Height | 視点の高さを調整するサブメニュー（High〜Low） |
| Zoom | ズーム倍率を調整するサブメニュー（Near〜Far、追従視点時のみ有効） |

**その他のメニュー項目**：

| 項目 | 説明 |
| ------ | ------ |
| Trajectory: Tail / Lane | 解曲線の表示方法を切り替える |
| Pause / Restart | アニメーションの一時停止・再開 |
| Repeat: Off→On / On→Off | アニメーション終端で自動終了するかどうかを切り替える |
| Quit | プログラムを終了する |

### 3.4 視点モードの説明

| モード | 説明 |
| -------- | ------ |
| Full view（`sws=0`） | 固定した真上からの俯瞰。アトラクタ全体の形状を確認できます。 |
| Follow the Plane（`sws=1`） | 紙飛行機の斜め後方に追従します。`u`/`d` で上下、`+`/`-` でズームを調整できます。 |
| From the Center（`sws=2`） | アトラクタ中心の球から紙飛行機を見上げる視点です。 |
| From Outside（`sws=3`） | 外側から中心方向を見る視点です。 |

## 4. アルゴリズムと処理手順

### 4.1 Rössler 方程式の数値積分

プログラムの中核は `solv_de()` 関数による常微分方程式の数値積分です。使用している方程式は標準的な Rössler 方程式を時間再スケーリングしたもので、以下の形をしています。

$$\dot{x} = (2.2 - z)(-y - 20z)$$
$$\dot{y} = (2.2 - z)(x + 0.2\,y)$$
$$\dot{z} = (2.2 - z)(0.01 + z(x - 6.7))$$

時間刻み $\Delta t = 0.005$ のオイラー法で離散化し、6000 ステップ分の軌道点を配列 `de_sol` に保存します。

### 4.2 局所座標フレームの計算

紙飛行機を曲線上に正しく配向させるため、解曲線に沿って Frenet フレームに相当する局所座標系を計算します。

- **接線方向**：連続する 2 点の差分 $\mathbf{v}_2 - \mathbf{v}_1$
- **法線方向 `de_n`（アトラクティブ方向）**：  
  主軌道 $\mathbf{v}$ に隣接した補助軌道 $\mathbf{u}$ を同時に積分し、接線ベクトルと変位ベクトル $\mathbf{u}_2 - \mathbf{v}_2$ の外積として求めます。アトラクタが引き寄せる方向に対応します。
- **従法線方向 `de_a`（エクスパンディング方向）**：  
  接線ベクトルと法線ベクトルの外積として求めます。アトラクタが広がる方向に対応します。

補助軌道 $\mathbf{u}$ は各ステップ後に主軌道から `de_a` 方向に一定距離 0.2 だけ離れた位置へ再設定し、安定して局所フレームを追跡し続けるようにしています。数値積分の最初の 100 ステップは初期過渡状態として破棄します（空回し）。

### 4.3 3D シーンの描画

各フレームは以下の要素で構成されます。

| 関数 | 描画内容 |
| ------ | ---------- |
| `display_lane()` | 解曲線を帯（クワッド列）として描画。法線方向 `de_n` に垂直な `de_a` 方向に幅を持たせ、高さ z や x−y の値に応じて色付けします。 |
| `display_tail()` | 紙飛行機直近 50 点を折れ線（軌跡の尾）として描画します。 |
| `display_plane()` | 局所フレーム（`de_n`・`de_a`・接線）を利用して紙飛行機ポリゴンを配置・描画します。 |
| `display_sun()` | 原点付近に太陽を模したソリッド球を描画します。 |

`idle()` コールバックが毎フレーム呼ばれるたびに `time` を 1 増やし、参照する軌道点を進めることでアニメーションを実現しています。

### 4.4 立体視

`STEREO` マクロで立体視方式を選択します。

| 定数 | 方式 |
| ------ | ------ |
| `NONE` | 通常の単眼表示 |
| `QUADBUF` | クワッドバッファステレオ（シャッターメガネ等の専用ハードウェアが必要） |
| `BARRIER` | 視差バリア方式。ステンシルバッファを使って奇数列と偶数列のスキャンラインに左右の視点映像を交互に書き込みます。 |

`BARRIER` モードでは `reshape()` でステンシルバッファに縦縞マスクを一度だけ描画し、`display()` で左右の視点を切り替えながら同一シーンを計 4 回描画します。左右の視差は定数 `PARALLAX = 0.2`、収束距離は `DISTANCE = 8.0` で設定します。

### 4.5 視点モード

`sws` 変数で 4 種類の視点を切り替えられます。

| `sws` | 視点 |
| ------- | ------ |
| 0 | 真上からの固定俯瞰 |
| 1 | 紙飛行機の斜め後方から追従（デフォルト） |
| 2 | アトラクタ中心から紙飛行機方向を見る |
| 3 | 原点方向から外側へ向けた視点 |

視点 1 では `gluLookAt` の引数を局所フレーム（接線・`de_n`）から毎フレーム計算することで、紙飛行機が常に画面中央に来るように追従します。
